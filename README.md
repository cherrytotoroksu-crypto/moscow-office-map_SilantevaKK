# Карта офисного рынка Москвы

Интерактивная карта: тепловая раскраска по ценам/объёмам + слой зданий-точек.
Деления: административные границы, автономные округа, зоны (кольца), субрынки,
деловые районы (сформированные / формирующиеся).

## Как добавить новый квартал

1. Выгрузи здания нового квартала в файл `data/buildings_ГГГГQN.json`
   (например `data/buildings_2026Q2.json`). Формат — как у `buildings_2026Q1.json`:
   массив объектов с полями `name, address, developer, cls, year, status,
   on_sale, gba, gla, volume, price, zone, submarket, biz, lat, lng`.

2. Открой `index.html`, найди блок `const QUARTERS = [` (в начале скрипта)
   и допиши новую строку **сверху** (новые кварталы — первыми):

   ```js
   const QUARTERS = [
     { id: '2026Q2', label: 'Q2 2026', file: 'data/buildings_2026Q2.json' },
     { id: '2026Q1', label: 'Q1 2026', file: 'data/buildings_2026Q1.json' },
   ];
   ```

3. Двойной клик по `обновить-сайт.bat` — данные уедут на сайт, появится в списке
   «Квартал». Прошлые кварталы остаются доступны.

## Локальный запуск

```
python -m http.server 3456
```
затем открой http://localhost:3456
(карта читает данные через fetch, поэтому нужен локальный сервер, а не просто
двойной клик по index.html).

## Данные

- `data/mo.geojson` — районы Москвы и агломерации (146)
- `data/ao.geojson` — автономные округа (12)
- `data/zones.geojson` — зоны-кольца (4)
- `data/submarkets.geojson` — субрынки (17)
- `data/buildings_*.json` — здания по кварталам

Геометрия деловых районов временно строится из координат зданий (выпуклые
оболочки) и на карте скрыта — показывается сводной таблицей. Когда появится
файл с настоящими полигонами деловых районов, его можно подключить.
